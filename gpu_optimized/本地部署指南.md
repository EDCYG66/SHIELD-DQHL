## 第一步：安装 Conda 环境

如果你本地已有 `tf212` 环境且版本一致，跳过此步。

```bash
# 创建环境（和服务器一致）
conda create -n tf212 python=3.10 -y
conda activate tf212

# 安装 CUDA 工具包（conda-forge 版本，不需要系统级 CUDA）
conda install -c conda-forge cudatoolkit=11.8 cudnn=8.9 -y
```

## 第二步：安装 Python 依赖

```bash
conda activate tf212

# TensorFlow 2.12（和服务器一致）
pip install tensorflow==2.12.0

# CuPy — GPU加速核心库
# RTX 3060 用 CUDA 11.x 版本
pip install cupy-cuda11x

# 其他项目依赖（如果还没装）
cd /你的路径/MyProject
pip install -r requirements.txt  # 如果有的话
```

## 第三步：验证安装

```bash
conda activate tf212
cd /你的路径/MyProject

# 1. 验证 TensorFlow 能看到 GPU
python -c "import tensorflow as tf; print(tf.config.list_physical_devices('GPU'))"
# 应输出: [PhysicalDevice(name='/physical_device:GPU:0', device_type='GPU')]

# 2. 验证 CuPy 能用
python -c "import cupy as cp; a = cp.ones(10); print('CuPy OK:', a.sum())"
# 应输出: CuPy OK: 10.0

# 3. 运行正确性验证
python -m gpu_optimized.validate
# 5个测试全部 PASS 才算成功
```

## 第四步：本地参数调整（重要！）

6GB VRAM + 16GB 内存，需要调小参数：

### 修改 `gpu_optimized/config.py`

```python
# ===== 本地 RTX 3060 Laptop 推荐配置 =====
GPU_TRANSFER_THRESHOLD = 8       # 不变
VEC_ENV_COUNT = 8                # 服务器16 → 本地8（内存限制）
BATCH_SIZE = 128                 # 服务器256 → 本地128（VRAM限制）
TRAIN_EVERY_K = 1                # 不变
TRAIN_UPDATES_PER_TRIGGER = 4    # 服务器8 → 本地4（减少GPU压力）
THREAD_POOL_WORKERS = 8          # 不变（8核）
REPLAY_CAPACITY = 30000          # 服务器40000 → 本地30000（内存限制）
CUPY_MEMPOOL_LIMIT_BYTES = 1 * 1024 ** 3  # 服务器4GB → 本地1GB
```

### 为什么要改

| 参数 | 服务器值 | 本地值 | 原因 |
|------|---------|--------|------|
| VEC_ENV_COUNT | 16 | 8 | 每个环境占 ~500MB 内存，16个=8GB，本地16GB总内存撑不住 |
| BATCH_SIZE | 256 | 128 | 训练batch占VRAM，6GB要省着用 |
| TRAIN_UPDATES_PER_TRIGGER | 8 | 4 | 减少每步训练次数，降低GPU负载 |
| REPLAY_CAPACITY | 40000 | 30000 | 每条样本约1KB，省内存 |
| CUPY_MEMPOOL_LIMIT_BYTES | 4GB | 1GB | 6GB VRAM，TF占~200MB，CuPy最多给1GB |

## 第五步：运行训练

```bash
conda activate tf212
cd /你的路径/MyProject

# 先小规模测试（确认没问题）
python -m gpu_optimized.launcher \
  --optimization-level 3 \
  --n-envs 4 \
  --cupy-mem-limit-gb 1.0 \
  --episodes 5 \
  --steps 100 \
  --seed 42 \
  [其他训练参数...]

# 正式训练

  --optimization-level 3 \
  --n-envs 8 \
  --cupy-mem-limit-gb 1.0 \
  --episodes 300 \
  --steps 900 \
  --seed 42 \
  [其他训练参数...]
```

## 常见问题

### Q: `import cupy` 报错 CUDA not found

```bash
# 检查 CUDA 是否安装
nvidia-smi
nvcc --version

# 如果没有 nvcc，conda 安装：
conda install -c conda-forge cudatoolkit=11.8 -y

# 设置环境变量（加到 ~/.bashrc）
export LD_LIBRARY_PATH=$CONDA_PREFIX/lib:$LD_LIBRARY_PATH
```

### Q: TensorFlow 看不到 GPU

```bash
# 检查 NVIDIA 驱动
nvidia-smi  # 应显示 RTX 3060

# 检查 cudnn
python -c "import tensorflow as tf; print(tf.test.is_built_with_cuda())"

# 如果 False，重装 TF
pip install tensorflow==2.12.0 --force-reinstall
```

### Q: 训练时 OOM（显存不足）

```bash
# 进一步减小参数
--n-envs 4              # 减少并行环境
--cupy-mem-limit-gb 0.5 # CuPy 只给 500MB
```

同时修改 `config.py`：
```python
BATCH_SIZE = 64          # 再减小
```

### Q: 训练速度没有明显提升

确认 `TF_FORCE_GPU_ALLOW_GROWTH=true` 生效：
```bash
# launcher.py 已自动设置，但如果直接运行其他脚本需要手动设：
export TF_FORCE_GPU_ALLOW_GROWTH=true
```
